# EffectsModule

## Призначення
Відображає візуальні аури на юнітах (бафи/дебафи). Синхронізує позиції ефектів із рухом цілей.

## Залежності
- `SceneObjectManager` — створює об’єкти типу `aura`, оновлює позиції та видаляє старі.
- `getUnitPositionIfAlive` — колбек для актуальних координат; при відсутності позиції ефект скидається.
- `effects-db.ts` — реєстр рендерерів для `VisualEffectId`.

## Життєвий цикл
- `initialize()` — без додаткової логіки.
- `reset()` / `load()` — повністю очищують активні аури.

## Основні методи
- `tick()` — підтримує позиції аур у синхроні з юнітами, прибирає ефекти мертвих цілей.
- `applyEffect(unitId, effectId)` — додає або перевіряє наявний ефект; якщо юніт зник, виклик проігнорується.
- `removeEffect(unitId, effectId)` — знімає конкретну ауру.
- `hasEffect(unitId, effectId)` — перевіряє, чи застосовано ефект.
- `clearAllEffects()` — видаляє всі аури без винятків.

## Збереження
Стан не зберігається — ефекти є похідними від поточного бою та бонусів.
