# EventLogModule

## Призначення
Збирає історію важливих подій (успішні проходження нових рівнів мап і здобуті ключові скіли) та передає її в UI через DataBridge.

## Залежності
- `DataBridge` — публікує список подій у `EVENT_LOG_BRIDGE_KEY`.
- `TestTimeModule` — дає поточний ігровий час (`timePlayedMs`) для фіксації часу події.

## Життєвий цикл
- `initialize()` — пушить поточний список подій.
- `reset()` — очищає історію та синхронізує місток.
- `load(data)` — відновлює події зі слоту та санітизує значення.
- `save()` — повертає поточний список подій для збереження.

## Основні методи
- `registerEvent(type, text)` — створює новий запис із реальним часом і часом у грі.
- `getEvents()` — повертає копію історії для внутрішнього використання.

## Збереження
Події серіалізуються у слоті як масив `events`.
